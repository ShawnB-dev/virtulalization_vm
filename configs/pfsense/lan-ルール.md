# pfSense LAN ルール  
**English version →** [lan-rules.md](../pfsense/lan-rules.md)

設定リファレンス

このドキュメントでは、VirtualBox 環境で使用している pfSense の LAN インターフェースに設定されたファイアウォールルールについて説明します。  
LAN インターフェースは Host-Only ネットワーク（`10.0.0.0/24`）に割り当てられ、RHEL サーバーなど内部ホストの通信を管理します。

---

## LAN インターフェース概要

- **インターフェース:** LAN  
- **IP アドレス:** `10.0.0.1/24`  
- **ネットワーク:** Host-Only  
- **用途:** server01 など内部 VM の通信  

---

## ファイアウォールルール

### 1. LAN → Any（デフォルト許可ルール）

| 項目 | 値 |
|------|------|
| アクション | Pass |
| インターフェース | LAN |
| プロトコル | Any |
| 送信元 | LAN net |
| 宛先 | Any |
| 説明 | LAN からの全トラフィックを許可 |

LAN 内部ホストが pfSense、ホスト OS、インターネット（NAT 経由）へアクセスできるようにする基本ルールです。

---

### 2. LAN → pfSense WebGUI

| 項目 | 値 |
|------|------|
| アクション | Pass |
| プロトコル | TCP |
| 送信元 | LAN net |
| 宛先 | LAN address |
| ポート | 443 |
| 説明 | pfSense WebGUI へのアクセスを許可 |

---

### 3. LAN → DNS

| 項目 | 値 |
|------|------|
| アクション | Pass |
| プロトコル | TCP/UDP |
| 送信元 | LAN net |
| 宛先 | LAN address |
| ポート | 53 |
| 説明 | pfSense への DNS クエリを許可 |

---

## 備考

- 本ラボでは LAN は「信頼されたネットワーク」として扱います。  
- 追加の VM を導入する場合は、必要に応じてルールを追加できます。
