# pfSense WAN ルール  

**English version →** [wan-rules.md](../pfsense/wan-rules.md)

設定リファレンス

このドキュメントでは、VirtualBox 環境で使用している pfSense の WAN インターフェースに設定されたファイアウォールルールについて説明します。  
WAN インターフェースは VirtualBox NAT から DHCP で IP を取得し、「信頼されていないネットワーク」として扱われます。

---

## WAN インターフェース概要

- **インターフェース:** WAN  
- **IP 割り当て:** DHCP（VirtualBox NAT）  
- **用途:** pfSense および LAN ホストの外部アクセス  

---

## ファイアウォールルール

### 1. WAN への全受信トラフィックをブロック（デフォルト）

| 項目 | 値 |
|------|------|
| アクション | Block |
| インターフェース | WAN |
| プロトコル | Any |
| 送信元 | Any |
| 宛先 | WAN address |
| 説明 | WAN への受信トラフィックをすべてブロック |

pfSense のデフォルト設定であり、本ラボでもそのまま使用します。

---

### 2. WAN → LAN のアウトバウンド通信（自動 NAT）

pfSense のデフォルト NAT により、以下が自動的に許可されます。

- LAN → WAN：許可  
- WAN → LAN：拒否（明示的に許可しない限り）  

本ラボでは追加の WAN ルールは不要です。

---

## 備考

- ポートフォワーディングは設定していません。  
- 外部からのサービス公開は行っていません。  
- 企業ネットワークの一般的な境界ファイアウォール構成に近い設定です。
