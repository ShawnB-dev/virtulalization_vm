# サーバーネットワーク設定 

**English version →** [server-network-config.md](../vm/server-network-config.md)

RHEL サーバー（server01）のネットワーク構成

このドキュメントでは、VirtualBox 環境における RHEL サーバーのネットワーク設定について説明します。

---

## ネットワークインターフェース概要

- **インターフェース:** `ens18`（VirtualBox のバージョンにより異なる場合あり）  
- **接続:** Host-Only アダプタ  
- **ネットワーク:** `10.0.0.0/24`  
- **ゲートウェイ:** pfSense LAN（`10.0.0.1`）  

---

## 推奨：静的 IP 設定

設定ファイル：
```markdown
/etc/sysconfig/network-scripts/ifcfg-ens18
```

例：
```markdown
TYPE=Ethernet BOOTPROTO=none NAME=ens18 DEVICE=ens18 ONBOOT=yes IPADDR=10.0.0.10 PREFIX=24 GATEWAY=10.0.0.1 DNS1=10.0.0.1 DNS2=8.8.8.8
```

設定反映：
```markdown
sudo nmcli connection reload sudo systemctl restart NetworkManager
```

---

## 動作確認
```markdown
ping 10.0.0.1      # pfSense LAN ping 8.8.8.8       # インターネット（NAT 経由）
```

期待される結果：

- RHEL → pfSense に到達可能  
- RHEL → インターネットに到達可能  

---

## 備考

- pfSense DHCP を有効にすれば DHCP も利用可能ですが、ラボの一貫性のため静的 IP を推奨します。  
- DNS は pfSense または外部 DNS を使用できます。
