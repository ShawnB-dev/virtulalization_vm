# 05 Packet Tracer トポロジー  
Cisco ルーティング／スイッチングシミュレーション

このセクションでは、Packet Tracer を使用したネットワークトポロジー構築について説明します。VirtualBox 環境が仮想化とファイアウォール構築に焦点を当てているのに対し、Packet Tracer は Cisco ネットワークの視覚的なシミュレーションとルーティング／スイッチングの学習に最適化されています。

両者は技術的には接続されていませんが、同時に構築することで企業ネットワークの内部 LAN レイヤーを概念的に再現します。

---

## トポロジー概要

このラボで使用する基本トポロジーは以下の通りです。
```markdown
[Router] --- [Switch] --- [PC1] --- [Switch] --- [PC2]
```


この構成により、以下の学習が可能です。

- ルーターのインターフェース設定  
- スイッチポート設定  
- VLAN（任意）  
- IP アドレッシング  
- LAN 内の接続性テスト  

---

## デバイス一覧

| デバイス | 数量 | 役割 |
|---------|------|------|
| ルーター | 1 | 内部 LAN のデフォルトゲートウェイ |
| スイッチ | 1〜2 | LAN 接続およびセグメンテーション |
| PC | 2〜3 | 接続性テスト用ホスト |

---

## IP アドレッシング計画（例）

- ルーター LAN：`192.168.1.1/24`  
- PC1：`192.168.1.10/24`  
- PC2：`192.168.1.11/24`  
- デフォルトゲートウェイ：`192.168.1.1`  

（実際のアドレスは自由に変更可能です。）

---

## ルーターの基本設定

例：
```terminal
enable configure terminal interface g0/0 ip address 192.168.1.1 255.255.255.0 no shutdown exit
```

---

## スイッチの基本設定

```terminal
enable configure terminal interface range f0/1 - 24 switchport mode access switchport access vlan 1 no shutdown exit
```


### VLAN の例（任意）
```terminal
vlan 10 name USERS exit interface f0/1 switchport access vlan 10
```

---

## PC の設定

各 PC に以下を設定します。

- IP アドレス  
- サブネットマスク  
- デフォルトゲートウェイ  

例：

- IP：`192.168.1.10`  
- マスク：`255.255.255.0`  
- ゲートウェイ：`192.168.1.1`  

---

## 接続性テスト

Packet Tracer のシミュレーションモードまたは CLI を使用してテストします。

```terminal
ping 192.168.1.1 ping 192.168.1.11
```

期待される結果：

- すべてのホストがルーターに到達可能  
- 同一 VLAN 内のホスト同士が通信可能
