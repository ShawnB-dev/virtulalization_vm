# 02 アーキテクチャ  

**English version →** [02-architecture.md](../docs/02-architecture.md)


VirtualBox + Packet Tracer の統合プロジェクト構成

このプロジェクトは、2 つの独立した環境で構成されていますが、概念的には小規模企業ネットワークの主要コンポーネントを表現しています。

- **VirtualBox**：ファイアウォールとサーバーレイヤー  
- **Packet Tracer**：内部 LAN のルーティング／スイッチングレイヤー  

技術的には接続されていませんが、ネットワーク全体像を理解するための 1 つのプロジェクトとして設計されています。

---

## 概念的なネットワークモデル

```markdown
            インターネット
                  |
              [pfSense]
                  |
     -------------------------------
     |                             |
VirtualBox LAN             （概念上の）内部 LAN （Packet Tracer）
```


---

## VirtualBox 環境のアーキテクチャ

```markdown

+------------------------------------------------+ |                VirtualBox ホスト               | +------------------------------------------------+ |                         | pfSense VM                RHEL Server VM | Host-Only LAN (10.0.0.0/24)
```

### コンポーネント

- **pfSense VM**  
  - ファイアウォール／ルーター  
  - WAN（NAT）＋ LAN（Host-Only）  

- **RHEL Server VM**  
  - テスト用サーバー  
  - pfSense LAN に接続  

---

## Packet Tracer 環境のアーキテクチャ

```markdown
[Router] --- [Switch] --- [PC1] --- [Switch] --- [PC2]
```


### コンポーネント

- **Cisco ルーター**  
  - サブネット間ルーティング  
  - 静的／動的ルーティングの練習  

- **Cisco スイッチ**  
  - LAN 接続  
  - VLAN セグメンテーション（任意）  

- **PC／サーバー**  
  - 内部ホスト  
  - 接続性テスト用  

---

## VirtualBox VM 定義

| VM 名 | OS | CPU | RAM | Disk | 役割 |
|-------|------|------|------|--------|--------|
| pfSense | pfSense CE | 2 | 2GB | 10GB | ファイアウォール／ルーター |
| server01 | RHEL 9 | 2 | 2–4GB | 20GB | 内部テストサーバー |

---

## ネットワークアドレッシング概要

### VirtualBox LAN
- ネットワーク: `10.0.0.0/24`  
- pfSense LAN: `10.0.0.1`  
- RHEL: `10.0.0.x`  

### Packet Tracer LAN（例）
- ルーター LAN: `192.168.1.1/24`  
- PC1: `192.168.1.10`  
- PC2: `192.168.1.11`  

---

## 2 つのラボの関係性

VirtualBox と Packet Tracer は独立していますが、企業ネットワークの異なるレイヤーを表現しています。

- **VirtualBox**：境界（ファイアウォール）＋サーバー  
- **Packet Tracer**：内部 LAN（ルーティング／スイッチング）  

これにより、ネットワークの全体像を理解しながら、各レイヤーの役割を学習できます。
