# 06 ネットワーク統合  

**English version →** [06-network-integration.md](../docs/06-network-integration.md)


VirtualBox と Packet Tracer が 1 つのプロジェクトを構成する仕組み

VirtualBox と Packet Tracer の環境は技術的には接続されていませんが、同時に構築することで企業ネットワークの異なるレイヤーを表現し、1 つの総合的な学習プロジェクトとして機能します。

---

## 概念的なネットワークモデル

このプロジェクトは、典型的な企業ネットワークの構造を概念的に再現しています。
```markdown
インターネット → pfSense ファイアウォール → 内部 LAN（Packet Tracer）
```

### VirtualBox レイヤー（実際の VM）
- pfSense（境界ファイアウォール）  
- RHEL サーバー  
- Host-Only LAN  

### Packet Tracer レイヤー（内部 LAN シミュレーション）
- Cisco ルーター  
- スイッチ  
- PC／サーバー  

---

## なぜ 2 つの環境を使うのか？

### VirtualBox の強み
- 実際の OS を使用  
- 本物のファイアウォール挙動  
- NIC やルーティングテーブルのリアルな動作  
- WebGUI を含む pfSense の完全な機能  

### Packet Tracer の強み
- Cisco ネットワークの視覚的シミュレーション  
- VLAN、STP、ルーティングプロトコルの学習  
- 物理機器なしで高速に構築  
- トポロジーの変更が容易  

両者を組み合わせることで、現場で求められる広範なインフラ知識を実践的に習得できます。

---

## レイヤー間の概念的なつながり

技術的には接続されていませんが、以下のように役割が対応しています。

- **pfSense**：境界ファイアウォール  
- **Packet Tracer ルーター**：内部ゲートウェイ  
- **Packet Tracer スイッチ**：アクセスレイヤー  
- **Packet Tracer PC**：内部ホスト  

概念モデル：

```markdown
[インターネット] | [ファイアウォール / pfSense] | [コアルーター / Packet Tracer] | [スイッチ] | [ホスト]
```

---

## 統合によって得られるスキル

- レイヤードネットワーク設計の理解  
- 仮想化環境とネットワークシミュレーションの両方を扱う能力  
- ファイアウォール配置とセグメンテーションの理解  
- ルーティング／スイッチングの基礎  
- 複雑な環境を文書化するスキル
